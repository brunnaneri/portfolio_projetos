Projetos de Machine Learning

Previsão de Vendas - Lojas Rossmann

Objetivo

Desenvolver um modelo que caracterize bem o comportamento de venda das 1115 lojas da rede de Farmácias Rossmann incluídas na base de dados, o qual consiga fazer predições bem sucedidas das vendas de cada loja com uma antecedência de seis semanas.

Processo e Produto Final

As predições podem ser acessadas via celular, através de um BOT no app Telegram, que retorna as predições após a consulta do algoritmo, posto em produção, via API. O usuário deve repassar o número da loja, 'Store ID, para que receba a predição das vendas desta loja para as próximas seis semanas.

Resultados Financeiros

A taxa média de erro obtida com o modelo foi de cerca de 10%, resultado que melhora em duas vezes as previsões de venda feitas pelo time de negócio, visto que, o modelo utilizado anteriormente, baseado num modelo de média, gerava um erro de 20% nas predições.

Com o modelo implementado, foram calculados diferentes cenários que preveem o faturamento destas lojas para as seis semanas seguintes. Estes cenários foram calculados considerando o erro MAE do modelo, de forma que, no pior cenário o faturamento total das lojas será de $285,802,068.49 e no melhor de $287,339,116.7.

  • Python, Pandas, Numpy e Seaborn.
  • Jupyter Notebook.
  • Algoritmos de Regressão (Linear Regression, Linear Regularized Regression, Random Forest Regressor, XGBoost Regressor).

Sistema de Rankeamento de Clientes - Cross-Selling de Seguros

Objetivo

desenvolver um modelo que fosse capaz de rankear a base de clientes de uma seguradora de saúde por propensão a adquirir um segundo seguro, no caso, o de automóvel.

Processo e Produto Final

O produto final foi disponibilizado em uma planilha no Google Sheets que contém um botão, emplementado pelo Google Scripts, que quando pressionado consulta o algoritmo do modelo final via API, e retorna a lista rankeada por uma coluna de propensity_score, que corresponde a probabilidade de compra do seguro de automóvel para cada cliente da base repassada para o modelo.

Resultados Financeiros

No cenário em que se tem uma base de 76222 clientes calcula-se que o retorno obtido com o modelo é de $206593713.48, enquanto que utilizando a lista aleatória $75261102.84, de gera de 2.75x mais receita.

A solução desenvolvida otimiza a prospecção de novos clientes pelo time de negócio, visto que, a lista rankeada pelo modelo é cerca de 3x superior que uma lista aleatória.

Desse modo, no cenário estudado de uma base de 127 mil clientes, em 20 mil ligações feitas pelo time de vendas, utilizando a lista rankeada pelo modelo tem-se que 47% do interessados desta base estão incluídos nas 20 mil ligações, enquanto que, numa lista aleatória apenas 15,75% dos interessados estariam dentro dessas 20 mil ligações.

As ferramentas utilizadas foram:

  • Python, Pandas, Numpy e Seaborn.
  • Jupyter Notebook.
  • Algoritmos de Classificação (KNN Classifier, Logistic Regression, XGBoost Classifier, Random Forest Classifier, Extra Trees Classifier.
  • Pacotes de Machine Learning sklearn e xgboost.
  • BorutaPy (seleção de atributos).
  • Flask - Python API's.
